http-equiv 属性为名称/值对提供了名称。并指示服务器在发送实际的文档之前先在要传送给浏览器的 MIME 文档头部包含名称/值对。当服务器向浏览器发送文档时，会先发送许多名称/值对。虽然有些服务器会发送许多这种名称/值对，但是所有服务器都至少要发送一个：content-type:text/html。这将告诉浏览器准备接受一个 HTML 文档。使用带有 http-equiv 属性的 标签时，服务器将把名称/值对添加到发送给浏览器的内容头部。

Phantomjs

PhantomJS是通过ghostdriver来支持Webdriver API的，所以如果是某个动作有问题的话，可以看看ghostdriver这里有没有issue。PhantomJS虽然是Headless，但是他的“渲染”核心是QtWebkit，JS引擎是JavaScriptCore，而不是FF的Gecko。所以这些差异都可能是造成动作无效的原因。

模拟浏览器容器

网站为了复用框架（C/S 的模式上，后端只提供 API，这样无论 Web 和 APP 都能共用一套 API ）或者是处理异步请求或者精致渲染都需要用js渲染，这为爬虫带来了很多困难。phantomjs可以从行为上模仿浏览器，从而拿到静态网页上拿不到的数据。

  1. 用普通的phantomjs方法，就可以抓到简单的js渲染数据了，比如淘宝之类的。可以通过截图来观察phantomjs渲染了多少出来。
  2. 还有一部分变态的网站通过简单的phatomjs借口无法渲染出来，这就需要等phantomjs渲染，直到渲染完成再进行抓取内容，典型的网站有xx文书网的搜索页面。本来想在这里贴代码的，但博主要切jj,只好作罢，大家可以根据链接找到原文5。
